# SLDAgent 桌面端原型设计

> 版本：v1.0  
> 日期：2026/06/16  
> 输出位置：`Document/UX/`

---

## 1. 设计方向

### 概念：「暗色制图工作台 / Dark Cartographic Workbench」

SLDAgent 是一个面向 GIS 专业人员与开发者的**样式编辑器**。界面需要同时承载三层信息：

1. **结构层**：SLD 的树形层级
2. **属性层**：复杂符号参数的编辑
3. **代码层**：GeoStyler JSON / SLD XML 的实时映射

因此采用**高密度、低干扰、强聚焦**的暗色工作台美学：

- **深色主背景**降低长时间编辑的视觉疲劳
- **高饱和 accent**（青绿 + 琥珀）用于选中态、地图元素、AI 提示
- **等宽字体 + 人文无衬线**的组合，区分代码与界面文案
- **微妙的网格/经纬线纹理**呼应制图主题
- **三栏固定布局**，配合标签页与折叠面板管理信息密度

## 2. 设计语言系统

### 2.1 色彩

```css
:root {
  --bg-primary: #0d1117;       /* 主背景 */
  --bg-secondary: #161b22;     /* 面板背景 */
  --bg-tertiary: #1c2128;      /* 输入框、卡片 */
  --bg-hover: #21262d;         /* hover */
  --bg-active: #30363d;        /* active / selected */

  --border-default: #30363d;
  --border-subtle: #21262d;

  --text-primary: #e6edf3;
  --text-secondary: #8b949e;
  --text-tertiary: #6e7681;
  --text-disabled: #484f58;

  --accent-teal: #2dd4bf;      /* 主强调色：选中、地图、成功 */
  --accent-teal-dim: #115e59;
  --accent-amber: #f59e0b;     /* 次强调色：AI、警告、高亮 */
  --accent-amber-dim: #78350f;
  --accent-red: #f87171;       /* 错误 */
  --accent-blue: #60a5fa;      /* 信息 */
  --accent-purple: #a78bfa;    /* 代码关键字 */
}
```

### 2.2 字体

| 用途 | 字体 | 说明 |
| --- | --- | --- |
| 界面标题 / 品牌 | **Space Grotesk** | 几何感强，有制图工具的专业气质；以本地 TTF 文件加载，保证离线可用 |
| 正文 / 标签 | **IBM Plex Sans** | 清晰易读，技术感适中；以本地 TTF 文件加载 |
| 代码 / JSON / XML | **JetBrains Mono** | 等宽，连字支持，开发者友好；以本地 TTF 文件加载 |
| 树节点图标 | **Phosphor Icons** | 线条图标，与暗色主题协调 |

字体文件位于 `SourceCode/frontend/public/fonts/`，通过 `fonts.css` 以 `@font-face` 声明。Tailwind 配置中应将 `--font-sans`、`--font-mono` 与品牌字体分别映射到 `IBM Plex Sans`、`JetBrains Mono`、`Space Grotesk`。

### 2.3 间距与圆角

- 基础单位：4px
- 面板内边距：16px / 20px
- 组件间距：12px
- 卡片圆角：8px
- 输入框圆角：6px
- 按钮圆角：6px
- 标签页圆角：6px 6px 0 0

### 2.4 层级与阴影

- 主窗口无阴影（Electron 原生窗口）
- 浮动面板：`0 8px 24px rgba(0,0,0,0.4)`
- AI 助手气泡：左侧发光边框 `box-shadow: 0 0 0 1px var(--accent-amber-dim)`

## 4. 左侧面板：SLD 树

### 4.1 树节点层级

```text
StyledLayerDescriptor v1.0.0
└── 📁 NamedLayer: 综合交通与设施图层
    └── 📁 UserStyle: 我的综合样式
        ├── 📝 Name: my_comprehensive_style
        ├── 📝 Title: 我的综合样式
        ├── 📝 Abstract: 该样式用于展示点状设施、线状道路和面状区域
        ├── ☑ IsDefault: true
        │
        ├── 📁 FeatureTypeStyle: 背景区域
        │   ├── 📝 Title: 背景区域
        │   ├── 📝 Abstract: 先绘制面状区域作为底图
        │   └── 📄 Rule: 区域面规则
        │       ├── 📝 Title: 居民区渲染
        │       ├── 🔍 Filter: geom_type = 'polygon'
        │       ├── 🔢 Scale: 1:1000 ~ 1:50000
        │       └── 🎨 Symbolizer: Polygon
        │
        ├── 📁 FeatureTypeStyle: 道路网络
        │   └── 📄 Rule: 主干道规则
        │       ├── 🔍 Filter: geom_type = 'line' AND road_class = 'highway'
        │       ├── 🔢 Scale: 1:2000 ~ 1:100000
        │       └── 🎨 Symbolizer: Line
        │
        └── 📁 FeatureTypeStyle: 设施点
            ├── 📄 Rule: 学校点规则
            │   ├── 🔍 Filter: geom_type = 'point' AND facility_type = 'school'
            │   ├── 🔢 Scale: 1:1000 ~ 1:20000
            │   ├── 🎨 Symbolizer: Point
            │   └── 🎨 Symbolizer: Text
            │
            └── 📄 Rule: 其他设施点规则
                ├── 🚫 ElseFilter: 兜底规则
                ├── 🔢 Scale: 1:1000 ~ 1:50000
                └── 🎨 Symbolizer: Point
```

### 4.2 节点视觉规范

| 节点类型 | 图标 | 颜色 | 说明 |
| --- | --- | --- | --- |
| StyledLayerDescriptor | 🗺️ | --text-secondary | 根节点，只读 |
| NamedLayer | 📁 | --text-secondary | 图层 |
| UserStyle | 🎨 | --accent-teal | 样式容器 |
| FeatureTypeStyle | 📂 | --accent-blue | 绘制顺序组 |
| Rule | 📄 | --text-primary | 规则单元 |
| Filter | 🔍 | --accent-amber | 筛选条件 |
| Scale | 🔢 | --text-secondary | 比例尺 |
| Symbolizer - Point | ⬤ | --accent-teal | 点符号 |
| Symbolizer - Line | ═ | --accent-teal | 线符号 |
| Symbolizer - Polygon | ▩ | --accent-teal | 面符号 |
| Symbolizer - Text | T | --accent-amber | 文字标注 |
| Symbolizer - Raster | ▦ | --accent-purple | 栅格 |
| ElseFilter | 🚫 | --accent-red | 兜底规则 |

### 4.3 交互

- **单击节点**：中间面板切换为对应属性编辑
- **双击节点 / 点击展开箭头**：折叠/展开子节点
- **右键节点**：上下文菜单（添加 Rule、添加 Symbolizer、删除、重命名）
- **拖拽节点**：调整 FeatureTypeStyle / Rule 的顺序
- **选中态**：左侧 3px 青色竖线 + 背景 `--bg-active`
- **Hover**：背景 `--bg-hover`

## 5. 中间面板：编辑 + 预览

### 5.1 标题栏

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ [≡]  SLDAgent      [📥 导入] [📤 导出]                                    │
└────────────────────────────────────────────────────────────────────────────┘
```

- 导入/导出作为全局文件操作放在标题栏。
- 「文件 / 视图 / 帮助」菜单当前冗余，已移除，聚焦核心功能。

### 5.2 属性编辑区与 LLM 问答

中间面板上半部分采用左右布局：

| 区域 | 宽度 | 说明 |
| --- | --- | --- |
| 属性编辑区 | 自适应 | 根据选中节点类型动态渲染对应表单 |
| LLM 问答面板 | 320px | 圆角卡片，与属性编辑区并排、间距紧凑；专门回答当前属性面板相关问题 |

#### LLM 问答面板

```text

│                         ┌─────────────────────────────┐        │
│                         │ 💬 属性问答              [v] │        │
│                         ├─────────────────────────────┤        │
│                         │ AI 助手                     │        │
│      属性编辑卡片        │ 我可以解答当前属性面板中...  │        │
│                         │                             │        │

│                         │ MinScaleDenominator 是什么？ │        │
│                         │                             │        │
│                         ├─────────────────────────────┤        │
│                         │ [ 询问当前属性... ] [➤]     │        │

```

- 与属性编辑区并列，以独立圆角卡片形式呈现，四周留有空隙，不再贴边。
- 默认展开，可折叠。
- 输入框 placeholder：`"询问当前属性..."`
- 发送后将问题追加到对话列表，实际实现时会把当前节点上下文一起发给后端 LLM。
- 与右侧面板的「AI 辅助」（规则解释 / 智能预警）定位区分：右侧面板聚焦**当前选中 Rule 的总结与预警**；本面板聚焦**属性参数的解释与用法**。

## 7. 关键交互状态

### 7.1 选中状态

- 树节点选中：左侧 3px 青色竖线 + `--bg-active` 背景
- 输入框聚焦：1px `--accent-teal` 边框 + 微弱发光
- 按钮 Hover：背景从 `--bg-tertiary` → `--bg-hover`
- 按钮 Active：背景 `--bg-active`，向下偏移 1px

### 7.2 错误状态

- 输入框错误：1px `--accent-red` 边框 + 红色提示文字
- 校验面板错误项：左侧红色竖线 + 红色图标
- 树节点错误：节点图标变红，悬停显示 tooltip

### 7.3 拖拽排序

- 拖拽时节点半透明，跟随鼠标
- 放置目标显示 2px 青色指示线
- 放置后触发 Store 更新，代码区实时刷新

## 8. 响应式策略

由于 Electron 窗口最小宽度建议 **1200px**，三栏布局在常规窗口下保持稳定。

- **窗口宽度 < 1200px**：右侧面板自动折叠为图标栏，点击图标展开
- **窗口宽度 < 900px**：中间编辑区与预览区上下堆叠
- **默认推荐窗口尺寸**：1400 × 900px

## 10. 已决策的设计问题

以下问题已在本阶段确定，作为 UI 实现依据：

1. **主题切换**：MVP 仅暗色主题，不实现亮/暗切换。
2. **AI 面板**：右侧面板底部独立卡片，琥珀色左边框，可折叠。
3. **树节点标签**：显示完整 XML 标签名（如 `sld:NamedLayer`），不额外显示 GeoStyler kind 或中文语义；类型通过图标与颜色区分。
4. **预览数据**：使用 `SourceCode/data/sample/` 内置本地样本数据（shapefile/GeoJSON）。
5. **属性面板分组**：支持按语义分组折叠，分组定义由 `symbolizer-schemas.json` 提供。
