SLDAgent — 模块与核心类设计

基于 requirements.md、technical-details-index.md、GeoStyler 复用报告 收敛出的高层模块划分与核心类。

1. 设计原则

  • 后端持有权威 Style 状态：前端把人工编辑作为 patch 提交，后端应用后返回新状态，保证多轮对话上下文一致。
  • JSON Schema 是唯一契约：LLM 输出、运行时校验、接口文档都围绕 Schema 展开。
  • GeoStyler 是 SLD 唯一权威：前后端使用同一版本 geostyler-sld-parser。
  • 失败即回退：后端保存 last_valid_style，生成或校验失败时直接回退，不维护多轮历史。

7. 关键场景数据流

7.1 自然语言生成新样式

sequenceDiagram autonumber participant U as 用户 participant F as Vue 前端 participant W as WsClient participant R as MessageRouter participant S as AgentSession participant K as KnowledgeBaseLoader participant L as LlmClient participant V as StyleParamsValidator participant B as StyleBuilder participant P as SldService U->>F: 输入“蓝色渐变虚线” F->>W: send({type:'generate', ...}) W->>R: route R->>S: handleGenerate S->>K: 加载 activeDomain 知识 S->>L: 注入 Prompt L-->>S: StyleParams (JSON) S->>V: validate(params) alt 校验失败 S-->>W: 返回错误 W-->>F: 显示错误 else 校验通过 S->>B: build(params) B-->>S: GeoStyler Style S->>P: writeStyle + XSD/roundtrip 校验 alt 校验失败 S->>S: rollback() S-->>W: 返回错误 else 校验通过 S->>S: currentStyle = newStyle
lastValidStyle = newStyle S-->>W: GenerationResult W-->>F: 更新 Inspector / Map / Rules end end

7.2 多轮增量修改

sequenceDiagram autonumber participant U as 用户 participant F as Vue 前端 participant W as WsClient participant S as AgentSession participant K as KnowledgeBaseLoader participant L as LlmClient participant B as StyleBuilder participant P as SldService U->>F: “把线宽改为 3” F->>W: send({type:'modify', instruction}) W->>S: handleModify S->>K: 加载 modification_rules S->>L: Prompt + 完整 StyleParams L-->>S: StyleParams（仅变更 + preserve） S->>B: build(params, currentStyle) B-->>S: newStyle S->>P: writeStyle + 校验 alt 失败 S->>S: currentStyle = lastValidStyle S-->>F: 错误 + 回退提示 else 成功 S->>S: lastValidStyle = newStyle
currentStyle = newStyle S-->>F: GenerationResult end

7.3 参数化精修提交（apply_patch）

sequenceDiagram autonumber participant U as 用户 participant E as RuleEditor participant ST as StyleStore participant W as WsClient participant S as AgentSession participant P as SldService U->>E: 在参数面板完成一组修改 E->>ST: 本地乐观更新（临时） U->>E: 点击“应用” E->>W: send({type:'apply_patch', patches}) W->>S: handleApplyPatch S->>S: applyPatches → newStyle S->>P: writeStyle + 校验 alt 失败 S-->>ST: 错误 ST->>ST: 撤销本地乐观更新 else 成功 S->>S: lastValidStyle = newStyle S-->>ST: GenerationResult ST->>E: 同步最新 Rule 状态 end

7.4 导入已有 SLD

sequenceDiagram autonumber participant U as 用户 participant F as Vue 前端 participant FS as FileService participant PS as SldParserService participant W as WsClient participant S as AgentSession participant P as SldService U->>F: 选择 .sld 文件 F->>FS: openFile FS-->>F: sldXml F->>PS: readStyle(sldXml) alt 解析失败 PS-->>F: 错误 F->>F: ValidationPanel 显示原因 else 解析成功 F->>W: send({type:'import_style', style}) W->>S: handleImport S->>P: writeStyle + 校验 S->>S: currentStyle = style
lastValidStyle = style S-->>F: GenerationResult F->>F: 更新 Map / Rules / Inspector end

8. 已落地设计与后续待定点

以下模块已在 design 文档中详细定义：

  • Filter 编辑器：详见 filter-editor.md。
  • 多轮修改协议：详见 agent-session.md §6。
  • xmllint 打包与降级：详见 xmllint-packaging.md 与 sld-service.md §6。

真正尚未定稿、需要后续详细设计的点：

  • 分类 / 分级无数据场景：LLM 直接给出断点 / 唯一值，还是拒绝生成？
  • 多 Symbolizer 混合：同一 Rule 中 Fill + Text / Line + Text 的组合规则与 Builder 支持。
  • 栅格样式（RasterSymbolizer）：是否作为独立 Builder，以及 colorMap、contrast 等字段的 Schema 定义。
  • Session 持久化：MVP 不持久化，后续是否保存当前 Style 与聊天历史到本地文件。
